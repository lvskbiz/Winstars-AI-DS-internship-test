# Task 2: NER + Image Classification Pipeline

This task contains two independent models and one orchestration script:

- a transformer-based NER model that extracts animal mentions from text;
- an image classification model that predicts the animal class in an image;
- a pipeline that compares both outputs and returns a boolean value.

## Expected structure

```text
Task 2/
├── eda.ipynb
├── image_classification/
│   ├── prepare_data.py
│   ├── inference.py
│   └── train.py
├── ner/
│   ├── prepare_data.py
│   ├── inference.py
│   └── train.py
├── pipeline.py
├── README.md
└── requirements.txt
```

## Data expectations

### Image classification

The image dataset should contain at least 10 animal classes and be split as:

```text
dataset/
├── train/
│   ├── bear/
│   ├── cat/
│   └── ...
└── valid/
    ├── bear/
    ├── cat/
    └── ...
```

### NER

Both `train-file` and `valid-file` must be JSONL files. Each line must look like:

```json
{"tokens": ["There", "is", "a", "cow", "in", "the", "picture", "."], "ner_tags": ["O", "O", "O", "B-ANIMAL", "O", "O", "O", "O"]}
```

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r "Task 2/requirements.txt"
```

## Training

Prepare NER data:

```bash
python3 "Task 2/ner/prepare_data.py" --output-dir data/ner
```

Prepare image dataset:

```bash
python3 "Task 2/image_classification/prepare_data.py" \
  --output-dir data/animals \
  --train-per-class 80 \
  --valid-per-class 20 \
  --test-per-class 20
```

Train NER:

```bash
python3 "Task 2/ner/train.py" \
  --train-file data/ner/train.jsonl \
  --valid-file data/ner/valid.jsonl \
  --output-dir artifacts/ner_model
```

Train image classification:

```bash
python3 "Task 2/image_classification/train.py" \
  --data-dir data/animals \
  --output-dir artifacts/image_model
```

## Inference

NER:

```bash
python3 "Task 2/ner/inference.py" \
  --model-dir artifacts/ner_model \
  --text "There is a cow in the picture."
```

Image classification:

```bash
python3 "Task 2/image_classification/inference.py" \
  --model-dir artifacts/image_model \
  --image-path samples/cow.jpg
```

Full pipeline:

```bash
python3 "Task 2/pipeline.py" \
  --text "There is a cow in the picture." \
  --image-path samples/cow.jpg \
  --ner-model-dir artifacts/ner_model \
  --image-model-dir artifacts/image_model
```

## Edge cases to document in the notebook

- plural animal mentions such as `dogs` or `wolves`;
- multiple animal mentions in one sentence;
- text with no animal entity;
- images that are difficult to classify or contain multiple animals.

## Quick experiment results

The repository now includes a small reproducible run using:

- `Rapidata/Animals-10` reduced to `30 train / 10 valid / 10 test` images per class;
- synthetic BIO-tagged NER data generated by `ner/prepare_data.py`;
- `distilbert/distilbert-base-uncased` fine-tuned for NER for 2 epochs;
- `resnet18` trained from scratch for 3 epochs on the reduced image dataset.

Observed outputs from the local run:

- NER example: `"There is a cow in the picture." -> ["cow"]`
- NER validation loss after epoch 2: `0.0005027`
- Image classifier test accuracy on the reduced subset: `0.27`
- Pipeline example:
  `"There is a butterfly in the picture." + data/animals/test/butterfly/butterfly_0000.jpg -> True`

The image classifier accuracy is a baseline obtained without pretrained weights because the local environment failed to download the torchvision checkpoint due SSL certificate issues. In a normal environment, rerunning with `--pretrained`, more images per class, and more epochs should improve the result substantially.
